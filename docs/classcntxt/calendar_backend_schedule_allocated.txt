File: calendar_backend_schedule_allocated.php

Purpose
- Variant of schedule API that focuses on accepted shifts/templates (accepted=1).

Primary flow
- GET actions mirror main schedule API but filter templates/repeat sources to accepted=1.
- GET default: returns shifts with accepted filtering logic and user visibility window.
- POST: same CRUD actions as main schedule API.

Inputs
- GET: action, prjsrc, clientid, employeeid.
- POST JSON: same fields as calendar_backend_schedule.php.

Outputs
- JSON arrays and action responses.

Data/DB
- Tables: shifting_allocation, shifting_allocation_table, uerp_user, project, designation, notifications.

Side effects
- Updates shifts and notifications.

Risks / Notes
- No explicit auth/CSRF protections.
